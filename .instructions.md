# Rummikub Counter – Android App Instructions

> Diese Datei ist die vollständige Spezifikation für einen KI-Agenten, um eine native Android-App zu erstellen.
> Alle Informationen zu Modell-Input/Output, Post-Processing und UI sind enthalten – es ist kein weiterer Kontext nötig.

---

## Ziel

Native Android-App (Kotlin), die Rummikub-Steine per Kamera oder Galerie-Bild erkennt und die Punktzahl berechnet.
Die Erkennung läuft **lokal auf dem Gerät** mit einem vortrainierten YOLOv8-Modell (ONNX Runtime). Keine Internetverbindung nötig.

## YOLO-Modell

### Export (auf dem PC vorbereiten)

Das bestehende PyTorch-Modell muss einmalig in ONNX exportiert werden:

```python
from ultralytics import YOLO

model = YOLO("backend/models/rummikub_yolo.pt")
model.export(format="onnx", imgsz=640, simplify=True)
# Erzeugt: rummikub_yolo.onnx
```

Die exportierte `rummikub_yolo.onnx` wird in `app/src/main/assets/` abgelegt.

### Modell-Details

- **Input-Name:** `images`
- **Input-Shape:** `[1, 3, 640, 640]` – 1 Bild, 3 Kanäle (RGB), 640×640 Pixel
- **Input-Typ:** `Float32`, normalisiert auf 0.0–1.0
- **Input-Reihenfolge:** CHW (Channel, Height, Width) – NICHT HWC!
- **Output-Name:** `output0`
- **Output-Shape:** `[1, 18, 8400]` – 8400 Kandidaten mit je 18 Werten
- **Output-Layout pro Kandidat (nach Transposition zu [8400, 18]):**
  - Index 0–3: `cx, cy, w, h` (Center-X, Center-Y, Breite, Höhe) – relativ zu 640×640
  - Index 4–17: Klassen-Confidences für die 14 Klassen
- **14 Klassen:** Index 0–12 = Steinwerte 1–13, Index 13 = Joker
- **Kein Softmax nötig:** Die Confidence-Werte sind bereits nutzbar
- **Confidence Threshold:** 0.25 empfohlen
- **NMS IoU Threshold:** 0.45 empfohlen
- **NMS muss in der App implementiert werden** (ONNX-Export enthält kein NMS)

### Klassen-Mapping

```kotlin
val CLASS_MAP = mapOf(
    0 to 1, 1 to 2, 2 to 3, 3 to 4, 4 to 5,
    5 to 6, 6 to 7, 7 to 8, 8 to 9, 9 to 10,
    10 to 11, 11 to 12, 12 to 13, 13 to null // Joker
```

### 2. Preprocessing (ImagePreprocessor.kt)

```kotlin
fun preprocess(bitmap: Bitmap): Pair<FloatArray, LetterboxInfo> {
    // 1. Letterbox: Seitenverhältnis beibehalten, auf 640x640 padden
    //    - Längere Seite auf 640 skalieren
    //    - Kürzere Seite zentrieren, Padding mit Grau (114/255f)
    //    - LetterboxInfo speichern: scale, padX, padY (für spätere BBox-Rückrechnung)

    // 2. Pixel zu CHW Float-Array konvertieren:
    //    floatArray[0 * 640*640 + y*640 + x] = red   / 255f   (R-Kanal)
    //    floatArray[1 * 640*640 + y*640 + x] = green / 255f   (G-Kanal)
    //    floatArray[2 * 640*640 + y*640 + x] = blue  / 255f   (B-Kanal)

    val floatArray = FloatArray(3 * 640 * 640)
    // ... Pixel iterieren und befüllen ...
    return Pair(floatArray, letterboxInfo)
}

data class LetterboxInfo(
    val scale: Float,      // Skalierungsfaktor
    val padX: Float,       // Horizontales Padding in Pixeln (auf 640er Bild)
    val padY: Float        // Vertikales Padding in Pixeln (auf 640er Bild)
)
```

### 3. ONNX-Inferenz (YoloDetector.kt)

```kotlin
class YoloDetector(context: Context) {
    private val env = OrtEnvironment.getEnvironment()
    private val session: OrtSession

    init {
        val modelBytes = context.assets.open("rummikub_yolo.onnx").readBytes()
        val options = OrtSession.SessionOptions().apply {
            // Optional: NNAPI für GPU-Beschleunigung
            // addNnapi()
        }
        session = env.createSession(modelBytes, options)
    }

    fun detect(inputArray: FloatArray): Array<FloatArray> {
        val shape = longArrayOf(1, 3, 640, 640)
        val tensor = OnnxTensor.createTensor(env, FloatBuffer.wrap(inputArray), shape)

        val results = session.run(mapOf("images" to tensor))
        // Output Shape: [1, 18, 8400]
        val rawOutput = (results[0].value as Array<Array<FloatArray>>)[0] // [18, 8400]

        tensor.close()
        results.close()

        // Transponieren: [18, 8400] → [8400, 18]
        return transpose(rawOutput)
    }

    private fun transpose(matrix: Array<FloatArray>): Array<FloatArray> {
        val rows = matrix.size        // 18
        val cols = matrix[0].size     // 8400
        return Array(cols) { j -> FloatArray(rows) { i -> matrix[i][j] } }
    }
}
```

### 4. Post-Processing (NmsProcessor.kt)

```kotlin
fun postProcess(
    predictions: Array<FloatArray>,  // [8400, 18] nach Transposition
    letterboxInfo: LetterboxInfo,
    origWidth: Int,
    origHeight: Int,
    confThreshold: Float = 0.25f,
    iouThreshold: Float = 0.45f
): List<DetectedTile> {
    val candidates = mutableListOf<DetectedTile>()

    for (pred in predictions) {
        // pred[0..3] = cx, cy, w, h  (auf 640x640)
        // pred[4..17] = class confidences (14 Klassen)

        // Beste Klasse finden
        var bestClassIdx = 0
        var bestConf = pred[4]
        for (c in 1..13) {
            if (pred[4 + c] > bestConf) {
                bestConf = pred[4 + c]
                bestClassIdx = c
            }
        }

        if (bestConf < confThreshold) continue

        // BBox: center → corner Format
        val cx = pred[0]; val cy = pred[1]; val w = pred[2]; val h = pred[3]
        var x1 = cx - w / 2f
        var y1 = cy - h / 2f
        var x2 = cx + w / 2f
        var y2 = cy + h / 2f

        // Letterbox-Padding rückrechnen → Original-Bildkoordinaten
        x1 = ((x1 - letterboxInfo.padX) / letterboxInfo.scale).coerceIn(0f, origWidth.toFloat())
        y1 = ((y1 - letterboxInfo.padY) / letterboxInfo.scale).coerceIn(0f, origHeight.toFloat())
        x2 = ((x2 - letterboxInfo.padX) / letterboxInfo.scale).coerceIn(0f, origWidth.toFloat())
        y2 = ((y2 - letterboxInfo.padY) / letterboxInfo.scale).coerceIn(0f, origHeight.toFloat())

        val isJoker = bestClassIdx == 13
        val number = if (isJoker) null else bestClassIdx + 1

        candidates.add(DetectedTile(
            number = number,
            confidence = bestConf,
            isJoker = isJoker,
            x = x1.toInt(), y = y1.toInt(),
            width = (x2 - x1).toInt(), height = (y2 - y1).toInt()
        ))
    }

    // NMS: Überlappende Boxen filtern
    return nms(candidates, iouThreshold)
}

private fun nms(boxes: List<DetectedTile>, iouThreshold: Float): List<DetectedTile> {
    val sorted = boxes.sortedByDescending { it.confidence }.toMutableList()
    val result = mutableListOf<DetectedTile>()

    while (sorted.isNotEmpty()) {
        val best = sorted.removeFirst()
        result.add(best)
        sorted.removeAll { iou(best, it) > iouThreshold }
    }

    return result.sortedBy { it.x } // Links → Rechts sortieren
}

private fun iou(a: DetectedTile, b: DetectedTile): Float {
    val x1 = maxOf(a.x, b.x)
    val y1 = maxOf(a.y, b.y)
    val x2 = minOf(a.x + a.width, b.x + b.width)
    val y2 = minOf(a.y + a.height, b.y + b.height)

    val intersection = maxOf(0, x2 - x1) * maxOf(0, y2 - y1)
    val areaA = a.width * a.height
    val areaB = b.width * b.height
    val union = areaA + areaB - intersection

    return if (union > 0) intersection.toFloat() / union else 0f
}
```

- Oben: Originalbild mit eingezeichneten Bounding Boxes (Canvas-Overlay)
  - Jede Box mit Klassen-Label und Confidence-Prozent
  - Verschiedene Farben je nach erkanntem Wert
- Mitte: Große Gesamtpunktzahl (`totalScore`)
- Unten: Scrollbare Liste der erkannten Steine (TileCards)
  - Pro Karte: Nummer (oder "Joker"), Confidence als Balken
- FloatingActionButton: "Neues Foto" → zurück zu CameraScreen

## ViewModel State

```kotlin
data class AnalysisUiState(
    val isLoading: Boolean = false,
    val result: AnalysisResult? = null,
    val originalBitmap: Bitmap? = null,
    val error: String? = null
)

class AnalysisViewModel(application: Application) : AndroidViewModel(application) {
    private val _uiState = MutableStateFlow(AnalysisUiState())
    val uiState: StateFlow<AnalysisUiState> = _uiState.asStateFlow()

    private val detector = YoloDetector(application)

    fun analyze(bitmap: Bitmap) {
        viewModelScope.launch(Dispatchers.Default) {
            _uiState.update { it.copy(isLoading = true, error = null) }

            val startTime = System.currentTimeMillis()
            val (inputArray, letterboxInfo) = ImagePreprocessor.preprocess(bitmap)
            val rawOutput = detector.detect(inputArray)
            val tiles = NmsProcessor.postProcess(
                rawOutput, letterboxInfo, bitmap.width, bitmap.height
            )
            val elapsed = System.currentTimeMillis() - startTime

            val totalScore = tiles.sumOf {
                if (it.isJoker) 20 else (it.number ?: 0)
            }

            _uiState.update {
                it.copy(
                    isLoading = false,
                    originalBitmap = bitmap,
                    result = AnalysisResult(
                        tiles = tiles,
                        totalScore = totalScore,
                        tileCount = tiles.size,
                        processingTimeMs = elapsed
                    )
                )
            }
        }
    }

    fun reset() {
        _uiState.update { AnalysisUiState() }
    }
}
```

## Häufige Fehlerquellen

| Problem | Ursache | Lösung |
|---------|---------|--------|
| Alle Detektionen falsch positioniert | Letterbox-Padding nicht rückgerechnet | `LetterboxInfo` korrekt anwenden (padX, padY, scale) |
| Bild rotiert / seitwärts | CameraX Rotation ignoriert | `imageProxy.imageInfo.rotationDegrees` auswerten und Bitmap rotieren |
| Keine Detektionen | Confidence zu hoch oder Input nicht normalisiert | Threshold auf 0.25 setzen, Pixel durch 255f teilen |
| Kanal-Reihenfolge falsch | BGR statt RGB oder HWC statt CHW | Android Bitmap ist ARGB – ignoriere Alpha, nutze R/G/B in CHW-Reihenfolge |
| App-Crash bei großen Bildern | Bitmap zu groß für Speicher | Bitmap vor Verarbeitung auf max ~2000px Seite skalieren |
| OrtException beim Session-Laden | Model-Datei nicht in assets/ | Sicherstellen dass `rummikub_yolo.onnx` in `app/src/main/assets/` liegt |
